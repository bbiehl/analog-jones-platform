name: Keep plans focused on what's asked
description: User prefers plans scoped to the immediate ask — don't bundle future work into current plans
type: feedback

When planning, only include what the user explicitly asks for. Don't bundle future phases or features into the current plan.

**Why:** User asked for routes and pages only but the plan kept including auth + user management from a prior discussion. They had to correct this twice.

**How to apply:** If a previous conversation topic was deferred ("let's come back to this later"), do not re-include it in the current plan. Keep the plan laser-focused on the stated request.
